Sign in / Sign up

/login · /signup

Account creation

Email + password (Argon2id hashed; OWASP-recommended parameters). Minimum 10 characters. The first user to sign up automatically becomes admin and inherits any pre-existing projects.

Sign in

Session cookie is HttpOnly, SameSite=lax, session-only (no Max-Age) — the cookie is dropped when you close the browser, so you log out on every quit. Server-side sessions also expire after 14 days as an upper bound.

Single-window enforcement

Opening this app in a second tab/window broadcasts a claim. The older tab receives it and auto-logs-out via a toast warning, preventing concurrent sessions from one browser.

Per-user project isolation

Every project carries a user_id. Listing, viewing, editing, deleting — all scoped. Another user cannot see or open your projects (returns 404 to hide existence).

Brokerages

per-project broker selection

Two supported brokers

Every project chooses one broker at creation time. Existing projects use Alpaca; new projects can pick either:

  • Alpaca — fully wired. Simple API key + secret. Paper and live endpoints. Used by all four agents today.
  • ETrade — Phase 1 scaffolding shipped. OAuth 1.0a dance works, credentials persist encrypted. The runner currently skips ETrade projects (with a one-time log line) until the Phase 2 adapter is complete.

Adding an Alpaca project

Dashboard → + New Project → pick Alpaca → paste your API key + secret → choose paper or live endpoint. Keys are stored Fernet-encrypted in trading_projects.alpaca_api_key / alpaca_secret_key.

Adding an ETrade project

Dashboard → + New Project → pick ETrade. You'll provide the Consumer Key + Consumer Secret you got from etrade.com/etx/ris/developer. After saving, the UI redirects to /etrade/connect:

  • Click Open ETrade authorization page — a new tab opens to ETrade.
  • Log in with your brokerage account → click Accept.
  • ETrade shows a 5-character verifier code on screen.
  • Paste it back into our form → access tokens are exchanged and stored encrypted.

Daily token renewal is needed (ETrade tokens expire at midnight ET). Phase 2 will add a scheduled job.

Default broker (per user)

Set in /account → Default Brokerage. Just changes which option is pre-selected in the New Project dialog. Doesn't affect existing projects.

How agents stay broker-agnostic

The BrokerClient ABC in execution/base.py defines a common interface. execution.get_broker(project) returns either AlpacaClient or ETradeClient based on the project's broker_type column. Phase 2 replaces direct AlpacaClient(project) calls with the factory everywhere.

Core Architecture

how the bot thinks

Four-agent pipeline

A LangGraph state machine cycles Scanner → Strategist → Guardrail → Executor per project on the configured loop interval. The Scanner pulls candidates, the Strategist (LLM) picks the contract, the Guardrail (deterministic) gates risk, and the Executor submits the order.

Multi-tenant runner

One worker per active project. Market hours hard-gated — outside RTH the worker sleeps until next_open. Each tenant has isolated state, settings, and Alpaca keys (encrypted at rest with Fernet).

Settings model

app_settings holds globals (LLM keys, backup dir). project_settings holds per-tenant strategy/risk params. Every setting is editable from the UI — nothing is hard-coded.

Always-Available Features

shared across every page

Project sub-nav (top of every project page)

Sticky bar showing project name, market-open badge, runner status badge, and quick-links: Performance, Analysis, Risk, Wheel History, Reliability, Intelligence, Outlook, Backtest, Export CSV, Notifications, Logs, and the Optimize button. Updates every 5 s.

Optimize button

Auto-tunes project settings based on your current Alpaca cash balance and a chosen strategy template. Tiny accounts ($<5k) get 100% concentration + cheap-ticker watchlist; large accounts get diversified caps. Click to open dialog → auto-preview → Apply.

Floating AI chat

Bottom-right 💬 button opens a context-aware chat. Uses your configured LLM (Anthropic or Gemini). Knows about the current project, can call tools (get positions, place trades). Voice input + speak-back optional.

Notification bell

Top-nav badge with unread count. Lists recent in-app notifications: trades, kill-switch breaches, anomalies, daily digests. Click to mark all read.

Toast notifications

Action results slide up from the bottom-center, auto-dismiss after 4.5 s, click to dismiss early. Four kinds: ok info warn error. Replaces all native browser alert() dialogs.

Custom confirmations

Destructive actions (delete, close position, etc.) open a styled in-app dialog instead of the browser's ugly confirm() popup.

Project Dashboard

/projects/{id}

Performance summary (top)

Four cards at the top:

  • Starting Balance — equity at first portfolio snapshot.
  • Current Equity — live from Alpaca.
  • Short-Term Gain · 7d — $ and %, colored green/rose.
  • Long-Term Gain · all-time — same.

Refreshes every 30 s alongside the equity curve.

Account cards

Live Cash, Buying Power, Equity, Portfolio Value — straight from Alpaca's account endpoint.

Pipeline visualization

The 4-node graph (Scanner → Strategist → Guardrail → Executor) lights up as each step completes for the latest cycle. Errors turn the node rose.

Equity curve

Pastel line chart of your equity over the last 30 days, drawn on a canvas from portfolio_snapshots. P10/P90 banding implicit in the y-axis range.

Cycles-per-minute chart

30-minute bar chart of bot cycle activity. Quick sanity check — flat means the bot is idle.

Open positions + contracts

Two side-by-side tables. Shows all Alpaca positions, not just DB-tracked ones (untracked rows get a * marker and a Close* button that uses Alpaca's market-close endpoint directly). Tracked rows use the standard Close.

Recent activity timeline

Auto-updating list of agent events with plain-English narratives — every scan, decision, risk gate, execution, take-profit, and closure.

Project Strategy & Risk Settings

Collapsed panel exposing every project_setting (~35 keys): delta bands, DTE bands, watchlist, concentration caps, take-profit %, dry-run flag, etc. Save each row inline.

Performance

/projects/{id}/performance

Period metrics

Trade count, win rate, total/avg P&L, best/worst, max drawdown — for any window (today, week, month, YTD, all-time).

Equity curve (larger)

Same data as the dashboard mini-chart, but full-width with axis labels. Toggle period.

Closed trades table

Every closed CSP/CC with entry/exit dates, days held, premium collected vs close cost, realized P&L, closure reason. Sortable.

Analysis

/projects/{id}/analysis

Strategy Attribution

Slices realized P&L by dimension: delta band (0.15-0.20, etc.), DTE bucket (0-3, 4-7, 8-14, …), phase (CSP/CC), ticker, day of week, underlying price range. Horizontal bars show total $, with confidence labels (low/med/high) based on trade count.

Per-ticker heatmap

One row per ticker you've traded with win rate, trade count, total P&L. Colored by P&L magnitude.

Risk

/projects/{id}/risk

Kill switches

Configurable hard stops: daily loss ($X or %), weekly loss, portfolio drawdown, max open contracts. When breached, the runner halts the project until you clear the kill.

Greeks aggregation

Total delta, gamma, theta, vega across all open option contracts. Refreshed on demand (greeks calls are expensive).

Concentration limits

Max % of buying power any single ticker may consume (max_concentration_per_ticker). Guardrail rejects trades that would exceed it.

Earnings calendar lookup

Type a ticker → see next earnings date from yfinance. The scanner blocks tickers with earnings within avoid_earnings_within_dte days.

Wheel History

/projects/{id}/cycles

Cycle tracker

Every wheel cycle from the first CSP sale → (optional) assignment → CC sales → close. Tracks accumulated premium, adjusted cost basis (entry strike − total premium), realized P&L per cycle.

Adjusted cost basis

The CC strike selector uses (entry strike − accumulated premium) as the minimum legal strike, ensuring you don't sell calls below your effective cost. Prevents premature unwind at a loss.

Reliability

/projects/{id}/reliability

Position reconciliation

Periodic job compares DB-tracked positions to live Alpaca state. Mismatches logged to reconciliation_log; auto-sync optional. Manual trigger available.

Automated DB backups

Daily SQL Server full backup to backup_dir at backup_hour_utc. Old .bak files pruned per backup_retention_days. Run-now and prune-now buttons.

Order lifecycle tracker

Every submitted order recorded in orders table with Alpaca order_id, status, fills, errors. Poll-button to refresh from Alpaca.

Prometheus metrics

/metrics exposes cycle counts, error counts, latencies in standard Prometheus text format.

Intelligence

/projects/{id}/intelligence

AI recommendations

Ask Gemini/Claude for parameter tuning advice based on your last 30 days of attribution. Returns 1-2 specific changes with rationale. Click Apply to commit. Scales validated against the project's tunable schema (no out-of-range values).

Anomaly detection

Statistical detector runs every 15 min. Flags spikes (last hour ≥ 3× baseline), droughts (silent agent that was busy), and error storms (≥ 10 errors in last hour). Severity-tagged anomalies push to your notification channels.

Strategy templates

Three curated presets — Conservative Wheel, Aggressive Income, IV-Disciplined — each defining ~20 settings. Apply with one click; settings go into project_settings.

Market Outlook

/projects/{id}/outlook

Top performers ranking

Union of (S&P top-50, your watchlist, traded tickers — capped at 120). Each ranked by composite score: P&L (20%) + win rate (15%) + 1M/3M/6M momentum (40%) + IV rank (25%). Click any row to see its 30/60/90-day outlook. Bars parallelized — ~5s for 100 tickers.

30 / 60 / 90-day prediction (hybrid)

Per-ticker forecast that combines:

  • Quantitative — lognormal projection fit to trailing year of daily log-returns. Shows P10 / median / P90 price bands, expected return %, prob(spot > now), annualized σ.
  • LLM narrative — Gemini writes one paragraph per horizon, grounded in the quant numbers + IV rank + wheel history + momentum. Returns direction (bullish/neutral/bearish) + confidence (low/med/high).

Cached 12 hours per (ticker, horizon).

Backtest

/projects/{id}/backtest

Run a backtest

Pick a ticker universe + a date range + starting capital. Simulator walks daily bars, simulates CSP/CC selection using the project's current strategy params, and reports final equity, total P&L, max drawdown, # of trades, win rate. Saves to backtest_runs.

Notifications

/projects/{id}/notifications

Multi-channel alerts

Add channels: Discord webhook, Slack webhook, SMTP email, generic webhook. Each has a kind-filter (e.g. only trades, only errors). Test-send button verifies setup.

Daily digest

Once-a-day summary dispatched to all enabled channels: P&L, trade count, open positions, biggest mover. Configurable hour (UTC).

In-app history

Every dispatched notification stored. Mark-all-read button. The top-nav bell shows unread count live.

Logs

/projects/{id}/logs

Plain-English event log

Every agent action narrated. Filter by node (Scanner / Strategist / Guardrail / Executor / TakeProfit / …), event type, or free-text search. Expand any row to see the raw JSON payload + narrative.

Trade journal CSV export

Top-nav "Export CSV" link downloads every closed contract + closed equity position as a single CSV. Useful for tax prep and external analysis.

Portfolio (cross-project)

/portfolio

Aggregated view

One table across every project you own: equity, unrealized P&L, monthly realized, trade count, win rate, totals row. Each ticker's per-project performance summed up.

Global Settings

/settings

LLM configuration

Choose provider (Anthropic or Google), set API keys, pick models (strategist vs chat can differ — keep a cheap one for cycles, a smarter one for chat).

Backup configuration

backup_dir, backup_hour_utc, backup_retention_days.

Default loop interval, scheduler controls

Global defaults that individual projects can override via loop_interval_seconds in their project_settings.

LLM Cost

/cost

Token + cost tracking

Every LLM call recorded with provider, model, input/output tokens, USD cost. Rolled up by day. Spot expensive cycles quickly.

Response cache + rate limiting

LRU cache (SHA-256 keyed, 5 min TTL, 512 entries) and sliding-window rate limiter prevent runaway spend. Cache stats endpoint shows hit ratio.
