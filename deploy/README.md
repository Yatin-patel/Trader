# nginx + SSL deploy bundle

This folder is generated by `publish.bat`. Copy it to the nginx server at
`192.168.1.102`, then run the bash script — it installs nginx, gets a Let's
Encrypt cert, drops in the site config, and enables auto-renewal.

## What's in here

| File | Purpose |
|---|---|
| `nginx_traderapp.conf` | Site config: 80 → 443 redirect, HSTS, proxy_pass to Windows app server, static-asset caching, WebSocket upgrade for chat |
| `install_on_nginx_server.sh` | Idempotent installer for the nginx server. Re-running it walks every step and skips anything already in place. |
| `README.md` | This file. |

## Prerequisites on the nginx server

1. **Ubuntu / Debian** with `apt`. The script will install `nginx` and `python3-certbot-nginx` if missing.
2. **Public DNS A record** for `traderapp.dyndns.org` pointing at the nginx server's public IP. Without this, Let's Encrypt's http-01 challenge will fail.
3. **Inbound ports 80 and 443 open** on the nginx server's public-facing firewall + your router/NAT if the box is behind one. 80 is required for Let's Encrypt's challenge; 443 is the actual site.
4. **Network reachability** from nginx server → Windows app server on port 8000. They're both on `192.168.1.x` so this should just work.

## Pre-requisites on the Windows app server

- `API_HOST=0.0.0.0` in `.env` (so the app listens on all interfaces, not just localhost). `publish.bat` will set this automatically.
- Windows Firewall rule allowing inbound TCP/8000 from the LAN. `publish.bat` adds this rule.
- The app must be running. Start it with `start_all.bat`.

## Quick deploy

From the Windows app server, scp this whole folder to the nginx server:

```powershell
scp -r deploy yourname@192.168.1.102:/tmp/trader-deploy
```

Then ssh in and run the installer:

```bash
ssh yourname@192.168.1.102
cd /tmp/trader-deploy
chmod +x install_on_nginx_server.sh
sudo EMAIL=you@example.com ./install_on_nginx_server.sh
```

The `EMAIL=` is what Let's Encrypt uses for expiry warnings (and also if the cert is ever revoked).

## What the installer does (6 idempotent steps)

| # | Step | Skips when |
|---|---|---|
| 1 | apt install nginx | nginx already installed |
| 2 | apt install certbot + python3-certbot-nginx | certbot already installed |
| 3 | Write bootstrap HTTP-only config + symlink + reload | Site enabled AND cert file exists |
| 4 | `certbot --nginx -d traderapp.dyndns.org` | Cert already exists (cron handles renewal) |
| 5 | Replace bootstrap with full production config (HTTPS, headers, proxy) | always replaces — content comes from `nginx_traderapp.conf` |
| 6 | Enable `certbot.timer` for auto-renewal | timer already scheduled |

Re-running on an already-configured server walks all 6 steps and just emits `[skip]` lines.

```bash
# Validate the config

# Watch live traffic
sudo tail -f /var/log/nginx/access.log
```

## Troubleshooting

**`certbot` says "challenge failed"**
DNS isn't pointing at the nginx server yet, or port 80 isn't reachable from the public internet. `traderapp.dyndns.org` must resolve to a public IP that hits this server.

**Site loads but FastAPI returns 502 Bad Gateway**
- The Windows app server isn't reachable. Test from the nginx server: `curl http://<windows-ip>:8000/login`.
- The Windows Firewall is blocking 8000.
- The app is binding to `127.0.0.1` only. Edit `.env` on the Windows server: `API_HOST=0.0.0.0`, restart with `start_all.bat`.

**Cookies / session aren't sticking**
The app sets cookies with `Secure` flag only if `request.url.scheme == "https"`. The nginx config sets `X-Forwarded-Proto: https`, but FastAPI needs to trust that header. Add this to your FastAPI startup if cookies misbehave:

```python
from starlette.middleware.proxy_headers import ProxyHeadersMiddleware
app.add_middleware(ProxyHeadersMiddleware, trusted_hosts="*")
```

**HSTS broke an http:// dev login**
HSTS is sticky in the browser cache for ~2 years. To clear during testing: open the site in a private/incognito window, OR clear HSTS for the host via `chrome://net-internals/#hsts`.

## Updating the config later

Edit `nginx_traderapp.conf` in the project, re-run `publish.bat` on the Windows server to regenerate the deploy folder, scp it over, and re-run `install_on_nginx_server.sh`. Step 5 always overwrites the config and reloads nginx, so this is the supported update path.
